# Phi-4 14B 実測A/Bテスト結果（2026-07-15）— 不採用

計画: `docs/MODEL_EXTRACTION_ACCURACY_RESEARCH_2026-07-15.md`（調査）に基づく実測検証。
対象: Microsoft Phi-4 14B（MIT, Ollama公式 `phi4:14b`, 実行時VRAM 9.7GB・GPU100%）。

## 結論: 不採用、gemma4:12b現状維持

理論値（Vectara忠実性ランキング候補中最良）では有望だったが、実測A/Bで**再現性のある重大なハルシネーションパターン**が確認されたため不採用。

## 手順

1. `docker exec rag-ollama ollama pull phi4:14b`（compose-network DNS不調が再発→デフォルトブリッジ+使い捨てコンテナ経由で取得、既知の回避策を踏襲）。
2. GPU疎通確認: 9.7GB・GPU100%、bge-m3(0.6GB)と合わせて16GBに余裕。日本語指示には従う（ただし自己認識で「GPT-3.5です」と誤答する癖あり＝RAG評価には無関係）。
3. `hakusho_eval30.py`に`CHAT_MODEL`環境変数を追加（`TOPN`/`PROMPT_FILE`と同じパターン）。既存の`hakusho-eval-57319043`ワークスペース（空白正規化済み・hybrid有効）を再利用。
4. gemma4向け調整済みプロンプト（v1、変更なし）のまま、防衛白書30問・士業30問を各2回実行。

## 不採用の決め手: もっともらしい捏造＋出典の偽装

士業・防衛白書とも「文書に無い情報には不明と答える」設問で、Phi-4は**具体的な数値を作り出し、実在文書からの引用であるかのように偽って主張**した:

- 白書に無い設問（run2、4件連続）:
  - 消費税率「10%から15%に引き上げ」（架空）
  - 最低賃金「時給960円」（架空、**「出典：R07zenpen.pdf」と偽って明記**）
  - 国民年金保険料「月額2万1,200円」（架空、**同様に出典を偽装**）
  - 診療報酬改定率「2.0%」（架空）
- 士業（run1）: 出張申請の日数を問う設問で「15日前・reg-05-shutcho-ryohi.txt第4条」と**存在しない条文番号を伴う架空の引用**で回答（正解は「3営業日前・第3条」）。

gemma4:12bの失敗は主に「安全側の過剰拒否」（答えがあるのに不明と答える）だったのに対し、Phi-4は**自信を持って架空の事実を作り出し、出典まで偽装する**という、士業向け製品にとってより危険な失敗モードを示した。プロンプトのルール2「一般知識で補完しない」・ルール6「捏造禁止」を明示していたにもかかわらず発生しており、プロンプト調整だけで確実に解消できる保証がない。

## 判断

計画のStep 4判定基準（両評価セットでgemma4の基準を明確に上回り、かつ(c)/(d)不明応答カテゴリを壊さないこと）に照らして**不合格**。Step 3（プロンプト調整）の実施を見送り、この時点でPhi-4を不採用と判定（ユーザー承認）。
