# 診断レポート: Expand-Archive展開後のinstall.ps1がPS5.1でハングする問題

作成日: 2026-07-10（Claude Code）
対象: `docs/WINDOWS_NATIVE_BUILD_VERIFY_RESULT_2026-07-09.md` で報告された問題

## 結論（TL;DR）

- **ファイル内容・zip・スクリプトに欠陥はない**。ハングは「PS5.1の`Expand-Archive`実行が書き込んだ特定のNTFSファイル実体」に紐づく**この開発機のローカル状態**（フィルタドライバのファイル単位の状態スタック）であり、成果物側の欠陥ではないと判断。
- ただし顧客環境での再発リスクをゼロとは言い切れないため、**配布物の展開手順は `tar.exe -xf`（またはエクスプローラー右クリック展開）を正式手順とし、PS5.1の`Expand-Archive`は使用禁止**とする。
- Ollamaバージョン異常（0.23.0表示）は**誤解**で、同梱exeは正しく0.31.2だった（後述）。

## 検証事実（すべてWSL interop経由でWindows側を直接実行して確認）

| # | 検証 | 結果 |
|---|---|---|
| 1 | 各所のinstall.ps1のsha256比較（リポジトリ原本 / ビルド出力フォルダ / Expand-Archive展開先 / zip内エントリ） | **全て同一**（8bc45b28…）。破損なし |
| 2 | Zone.Identifier（MOTW）・NTFS属性・ADSの比較 | 差なし（ローカル生成zipのためMOTWなし） |
| 3 | `Parser::ParseFile`（PS5.1）を展開先install.ps1に実行 | **7msで正常**（errors=0）→ parserの問題ではない |
| 4 | `-File`実行（PS5.1）を展開先install.ps1に実行 | **300秒超ハング**（バナー出力すらなし）。何度でも再現 |
| 5 | 同一ディレクトリ内に`cp`した複製（install2.ps1）を`-File`実行 | **正常**（同内容・同ディレクトリなのに動く） |
| 6 | 同展開ツリーの`stop.ps1`を実行 | **ハング**（Expand-Archiveが書いた別ファイルも同症状） |
| 7 | ビルド出力フォルダ（zip経由なし・同じ10万ファイル構成）のinstall.ps1 | **正常** → ディレクトリ規模・構成の問題ではない |
| 8 | 展開先install.ps1をrename後に実行 | **ハング継続** → パス・ファイル名ではなく**ファイル実体（FileID）に紐づく** |
| 9 | pwsh 7で同じファイルを`-File`実行 | **正常** → PS5.1のファイルオープン経路のみ影響 |
| 10 | 排他モード（FileShare.None）でのオープン | **成功** → ハンドルを掴んでいるプロセスは存在しない |
| 11 | `Rename-Item`コマンド自体 | プロセスはハングするが**rename操作は完了していた**（操作は通るがプロセスが固まる） |
| 12 | powershell.exeのゾンビプロセス確認 | なし |
| 13 | node.exe（同展開ツリー内）の実行 | 正常 |

## 原因の推定

事実1〜13から、**Expand-Archive（21分間で10万ファイル/7.4GBを書き込んだセッション）が作成したファイル実体に対し、フィルタドライバ（Windows Defenderリアルタイム保護またはWindows Searchインデクサ）のファイル単位の状態（oplock/lease/スキャン待ちレコード）が壊れた形で残留**し、PS5.1の共有モード（deny-write）でのオープンだけがlease break待ちで永久にブロックされる、と推定。pwsh 7は共有モードが緩いため衝突しない。

管理者権限がなくハンドル所有者の直接特定（openfiles/handle.exe）はできなかったため、コンポーネントの断定は保留（Defender > Search の順で疑い）。

## 製品への影響と対策

- **顧客影響は低い**と判断: 顧客はWSLを持たず、通常はエクスプローラー右クリックで展開する。本事象はこの開発機の特定セッション状態に紐づく。
- ただし防御的に以下を正式化:
  1. **展開手順は`tar.exe -xf`を正式とする**（Windows 10 1803+標準搭載。Codex・Claude双方の検証で正常動作を確認済み）。エクスプローラー右クリック展開を代替手段とする（次回実機検証で確認）。
  2. **PS5.1の`Expand-Archive`は配布手順で使用禁止**と明記（5GB超ファイルのzip64問題・今回の事象・21分の展開時間、の3重の理由）。
  3. インストール実行は`powershell.exe`（5.1）で問題ない（tar展開したファイルでは全スクリプトが正常動作）。

## 付記: Ollamaバージョン「0.23.0」問題の解明

`ollama --version`は**接続先サーバーのバージョンを1行目に表示する**仕様（クライアントと違う場合は`Warning: client version is X`を付す）。
検証したところ同梱exeは `Warning: client version is 0.31.2` を出力しており、**同梱バイナリは正しくv0.31.2**。
「0.23.0」はポート11434で応答したWSL Docker側Ollamaのサーバーバージョンだった。
→ `export-windows.ps1`のversions.lock生成をclient行優先に修正済み（本コミット）。取得assetの再ダウンロードは**不要**。
