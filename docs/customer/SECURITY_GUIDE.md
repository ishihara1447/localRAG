# セキュリティガイド

## 外部通信方針

LocalRAG は、顧客文書を外部（インターネット上のサービス）へ送信
しないことを最優先の設計方針としています。

- LLM 推論・embedding（ベクトル化）は、同梱の Ollama コンテナ上で
  実行されます。OpenAI・Anthropic・Google Gemini などの外部 LLM
  プロバイダーは、UI から選択できないだけでなく、バックエンド API
  側でも拒否されます（allowlist 方式）。
- Ollama コンテナは Docker の内部ネットワーク（`rag-internal`）に
  隔離されており、ホストや外部から直接アクセスできません。
  AnythingLLM からのみ到達可能です。
- `HF_HUB_OFFLINE=1` により、Hugging Face Hub への実行時アクセスを
  禁止しています。モデルコードやモデル重みが起動中に外部から
  動的取得されることはありません。

## 既定で無効化している機能

以下は、社内利用時の攻撃面・情報漏えいリスクを減らすため、既定で
無効化しています。管理者が必要と判断した場合のみ、リスクを理解
したうえで有効化してください。

| 機能 | 既定値 | 理由 |
|---|---|---|
| Telemetry（利用状況送信） | 無効 | 利用状況が外部（Mintplex Labs）に送信されるのを防止 |
| Swagger API ドキュメント（`/api/docs`） | 無効 | API仕様の外部露出・攻撃面の縮小 |
| Community Hub からの bundle / Agent Skill ダウンロード | 無効 | 未検証の外部コード実行を防止 |
| 外部Web検索Agent（SerpAPI/Tavily/Bing等） | 無効（APIキー未設定） | 検索クエリや文書内容が外部に送信されるのを防止 |
| ワークスペースのAPI経由削除 | 拒否（`WORKSPACE_DELETION_PROTECTION`） | 誤操作・スクリプトミスによる文書の消失を防止 |

## 顧客文書の保存場所

アップロードされた文書、抽出テキスト、ベクターデータ、チャット履歴は
すべてローカルディスク上の `anythingllm-storage/` に保存されます。
外部のクラウドストレージやSaaSには一切送信されません。

## API キー管理

- REST API を利用する場合は、AnythingLLM の Settings → API Keys
  から発行してください。
- API キーは、社内の秘匿情報管理ルールに従って保管してください
  （リポジトリやチャットへの平文貼り付けは避けてください）。
- 不要になった API キーは速やかに無効化してください。

## モデル・embedding モデルを変更する場合の注意

- **embedding モデルを変更した場合、既存文書は自動的には再
  ベクトル化されません。** 異なる embedding モデルで生成した
  ベクトルは互換性がないため、変更後は全文書の再アップロード・
  再 embedding が必要です。
- `trust_remote_code` を要するモデル（カスタムコードを含むモデル）
  を導入する場合は、事前にコードをレビューし、コミットハッシュを
  固定したうえで導入してください。実行時に Hugging Face から
  動的にコードを取得する運用は行いません。
