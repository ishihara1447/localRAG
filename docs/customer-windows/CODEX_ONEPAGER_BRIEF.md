## このブリーフの使い方（Codexへ）

以下の**4テーマ＝計4〜6枚**を、1枚ずつ独立した「一枚絵（インフォグラフィック）」として作成する。
- ①インストール・アンインストール（1枚）
- ②使い方・概要（1枚／**中心イラスト主役**）
- ③使い方・詳細（**最大3枚**に分割／各枚とも**中心イラスト主役**）
- ④制約・注意事項（1枚）

各枚に「目的／想定読者／タイトル／載せる要素と正確な文言・数値／レイアウト案／強調・アイコン案／トーン」を指定した。
**②と③はイラストを一枚絵の中心に大きく据え、それを説明する吹き出し／引き出し線を周囲に配置する構図にする**（文字ブロックの羅列にしない）。
**文言・数値はこのブリーフの値を正とする**（言い換えは可、数値・製品名・パス・URLは改変しない）。Claudeが作図後にレビューする。

### 全4枚に共通の指定

- **比率**: 16:9 横（既存の `assets/idea-illustrations/` と揃える）。印刷併用したい場合はA4縦も可だが、まず16:9で作る。
- **想定読者**: 士業（弁護士・税理士・社労士等）の**非エンジニア**。専門用語は最小限、そえ書きで補足。
- **トーン**: 「簡単・安全・所内で完結」を伝える、信頼感のある落ち着いた配色（紺・白・淡いグレー＋アクセント1色）。図解・アイコン主体、文字は詰め込みすぎない。
- **製品名表記**: 画面上は「OTE-RAG」。サブに「所内文書AI検索システム」等の日本語説明を添えてよい。
- **共通の正確表記**（4枚で統一）:
  - URL: `http://localhost:3001`（`https`ではなく`http`）
  - 既定インストール先: `C:\LocalRAG`
  - 3サービス: `LocalRAG-Server` / `LocalRAG-Collector` / `LocalRAG-Ollama`
  - AIモデル: 回答生成=**Qwen3-8B**（日本語LLM）、文書検索=**bge-m3**（embedding）
  - GPU要件: **NVIDIA GPU / VRAM 16GB以上（RTX 5070 Ti相当）**

## 一枚絵①：インストール・アンインストール方法

- **目的**: 導入担当（IT担当 or 提供元エンジニア）が、導入と撤去の全体像を一目で掴む。
- **タイトル案**: 「OTE-RAG 導入と撤去の流れ」

### 載せる要素

**A. 事前条件（左上に小さくチェックリスト）**
- OS: Windows 11（Windows 10 21H2以降も可）
- GPU: NVIDIA VRAM 16GB以上（RTX 5070 Ti相当）＋ドライバ導入済 ← アイコンで強調
- ディスク空き: 20GB以上
- 権限: 管理者権限（**インストール時のみ**）
- インターネット: **不要**（オフライン完結）← 目立たせる

**B. インストール手順（中央に左→右の4ステップ・フロー矢印）**
1. **受け渡し・展開**: 配布zipをPCへコピー → `tar.exe -xf` またはエクスプローラー「すべて展開」で展開（※`Expand-Archive`は使用禁止、と赤系の注意バッジ）
2. **管理者PowerShellで実行**: `install.ps1` を実行（既定: 先 `C:\LocalRAG`・ポート `3001`）
3. **自動処理（インストーラが行う）**: 事前チェック → チェックサム検証（**20〜40分“無表示”は正常**の吹き出し）→ ファイル/モデルのコピー → 設定生成・DB初期化 → **3サービス登録・起動**（自動起動・異常時自動再起動のアイコン）
4. **完了**: `http://localhost:3001` をブラウザで開く → 初回のみ管理者アカウント作成

**C. アンインストール（右下に小ブロック）**
- 管理者PowerShellで `uninstall.ps1`
- **既定はデータを残す**（`C:\ProgramData\LocalRAG\uninstalled-<日時>\storage` に退避、再インストールで復元可）
- 完全削除は `-RemoveData`（元に戻せない、の警告アイコン）

### レイアウト/強調
- 中央を「①受け渡し→②実行→③自動処理→④完了」の横一列フローに。各ステップは番号バッジ＋アイコン＋1行説明。
- 「オフライン完結」「管理者権限はインストール時だけ」「20〜40分の沈黙は正常」の3点を吹き出しで特に強調（問い合わせが多い/誤解されやすい点）。
- アンインストールは色を変えた別ブロックで下部に。

## 一枚絵②：使い方（概要）

- **目的**: 日々使う士業スタッフが「起動不要・ブラウザで開くだけ・所内で完結」を直感的に理解する。
- **タイトル案**: 「毎日の使い方はこれだけ」
- **構図の方針（重要）**: **中心に1枚の大きなイラスト（シーン）を主役として据え、その周囲に吹き出し／引き出し線で説明を配置する**。文字ブロックの羅列にしない。

### 🎨 中心イラスト（主役・画面中央に大きく）
- **描くシーン**: 士業（弁護士/税理士のような人物）が自席のデスクでノートPCに向かい、画面にOTE-RAGのチャット画面（日本語のQ&A）が映っている。手元／脇には紙の書類やファイルが数点。
- **PCを囲む「安心の囲い」**: PC全体を淡い円 or シールドで囲い、「このPCの中だけで完結」を視覚化。囲いの外にクラウド/インターネットのアイコンを置き、囲いとの間を**×印や斜線で遮断**（データが外に出ない表現）。
- **人物の表情**: 安心・スムーズに使えている雰囲気（難しそうにしない）。

### 中心イラストから引き出す説明（周囲の吹き出し3つ＝使う3ステップ）
- **① PCの電源を入れるだけ**（引き出し先: PCの電源/起動）: 起動・停止操作は不要。裏で自動的に動く（Windowsサービス）。※起動直後の初回回答は少し時間がかかる旨を小さく添える。
- **② ブラウザで開く**（引き出し先: 画面）: 同じPCのブラウザ（Edge等）で `http://localhost:3001` を開く。URLは読みやすく大きめに。
- **③ 質問する**（引き出し先: チャット欄）: ワークスペース（＝文書をまとめる箱。案件別・分野別）を選び、日本語で質問 → **出典付きで回答**。

### 下帯（3アイコンで安心ポイントを常時提示）
- 🔒 **データは外部に出ない**（クラウド送信なし・PC内で完結）
- 📄 **回答には出典が付く**（どの文書のどこか）
- ❓ **文書に無いことは「不明」と答える**（作り話をしない）

### 強調
- 主役はあくまで中心イラスト。周囲の吹き出しは短い見出し＋1行に留める。
- 最も目立つコピーは「**毎日の起動・停止は不要**」（クラウドSaaSにない“ローカルでも手間なし”の体験差）。

### 一枚絵③-1：文書を取り込んで質問する（メイン操作）

- **タイトル案**: 「文書を取り込んで質問する」

**🎨 中心イラスト（主役）**: データの流れの断面図（アイソメトリック）。左→右で——
1. 紙/PDF/Wordの**書類**が、
2. PC内の**「ワークスペース」という箱**に入り、
3. 中で**検索できる形に変換（embed）**され（歯車/変換モチーフ）、
4. 人物の**質問**に**出典タグ付きの回答**が吹き出しで返る。
- 全体をPCの輪郭（またはシールド）で囲い「すべてこのPC内で処理」を明示（②の“外に出ない”囲いを踏襲）。
- ワークスペースは案件別・分野別に複数持てることが伝わるよう箱を2〜3個並べてもよい。

**周囲の吹き出し**
- **アップロード手順**（引き出し先: 書類→箱）:
  1. ワークスペースを開く
  2. アップロードアイコン（書類マーク）から文書をドラッグ＆ドロップ／選択
  3. 「Move to Workspace」→「**Save and Embed**」で取り込み（数十秒〜数分）
  4. 取り込み後、チャットで質問
- **対応形式**バッジ: **PDF ・ Word(.docx) ・ テキスト(.txt)**／注意: 文字情報のない**スキャン画像だけのPDFは読み取れない場合あり**
- **回答の性質**（引き出し先: 回答の吹き出し）: 出典付きで回答／**文書に無いことは「不明」**（作り話をしない）
- **強調**: 「**Save and Embed で取り込む**」を操作の要として最強調。

### 一枚絵③-2：バックアップと復元（データの保全）

- **タイトル案**: 「大切なデータを守る（バックアップと復元）」

**🎨 中心イラスト（主役）**: PC内のデータ（文書・履歴の束）が**1つのバックアップzip**にまとまり、そこから**外部媒体（社内サーバー/外付けディスク）へコピー**される流れ。逆向きに「復元（zip→PCへ書き戻し）」の矢印も薄く添える。カレンダーに「週1」マークを添える。

**周囲の吹き出し**
- **バックアップ**（引き出し先: zip生成）:
  - `backup.ps1`（管理者PowerShell）／**週1回推奨・業務時間外に**
  - 実行中はサービス自動停止→完了後自動再開（数分は利用不可）
  - 保存先: `C:\ProgramData\LocalRAG\backups\localrag-backup-<日時>.zip`
  - AIモデル本体は対象外（配布物から復元できるため）
- **外部保管**（引き出し先: 外部媒体）: zipは**外部媒体にもコピー**（PC故障対策）。中身は機密文書と同じ扱いで保管
- **復元**（引き出し先: 戻り矢印）: `restore.ps1 -BackupZip <zip>`。**現在のデータは上書きされる**ため実行前に相談推奨（警告アイコン）
- **強調**: 「**週1バックアップ＋外部媒体にコピー**」。

### 一枚絵③-3：日常の管理（起動・停止・状態確認・保存場所）

- **タイトル案**: 「ふだんの管理と保存場所」

**🎨 中心イラスト（主役）**: PCの中で**3つのサービス（歯車）が自動で回っている**様子（`Server`/`Collector`/`Ollama`のラベル付き）。PCの電源投入で自動起動・異常時は自動再起動、を矢印で表現。手前に「フォルダ（保存場所）」のアイコン群。

**周囲の吹き出し**
- **起動・停止**（引き出し先: 歯車）: 通常は不要（自動復帰）。メンテ時のみ `start.ps1` / `stop.ps1`
- **状態確認**（引き出し先: 歯車）: `Get-Service LocalRAG-*` → **3つとも `Running` なら正常**
- **保存場所**（引き出し先: フォルダ群）:
  - 文書/履歴/検索データ/設定: `C:\LocalRAG\app\server\storage`
  - AIモデル: `C:\ProgramData\LocalRAG\models`／ログ: `C:\ProgramData\LocalRAG\logs`
  - これらのフォルダを**手作業で移動・削除しない**
- **誤削除防止**: ワークスペースの誤削除防止が既定ON（API経由の削除は拒否、削除は画面から明示操作）
- **強調**: 「**3サービスがRunningなら正常**」（トラブル時の第一確認）。

## 一枚絵④：制約・注意事項

- **目的**: 導入前後に必ず押さえるべき制約・禁止事項・安全根拠を1枚で確認できる。顧問先説明にも使える。
- **タイトル案**: 「使う前に知っておくこと（制約と安全のしくみ）」

### 載せる要素（3カラム）

**A. 動作要件・制約（左）**
- **GPU必須**: NVIDIA VRAM 16GB以上（RTX 5070 Ti相当）。**無い/不足のPCでは動作しない**（一般オフィスPCは要件外の可能性）← 最重要、赤系で強調
- 1台にインストールする買い切り型（同時に使うのは基本そのPC上のブラウザ）
- 回答前に「考える過程」を挟むモデルのため、難問は回答完了まで数分かかることがある
- 他ソフト（動画編集・ゲーム・別AI）がGPUを使うと大幅に遅くなる
- スキャン画像だけのPDFは内容を読み取れない場合がある

**B. 禁止・変更しないこと（中央）**
- zip展開に **`Expand-Archive` を使わない**（`tar.exe` かエクスプローラー展開）
- 同梱ソフトの**バージョンを勝手に更新・変更しない**（検証済みの固定構成、`versions.lock`）
- データ用フォルダを手作業で移動・削除しない
- ライセンス表示（`LICENSES\` / `NOTICE`）を削除しない
- ポート 8888 / 11435 は内部用で変更不可（3001は変更可）

**C. 安全のしくみ＝外部に出ない根拠（右・安心感を出す）**
- AI処理（Qwen3-8B / bge-m3）はすべて**PC内**（Ollamaは `127.0.0.1:11435` の内部通信のみ）
- 外部AI（OpenAI/Anthropic/Gemini）は**API内部の許可リストで遮断**（設定でも有効化不可）
- テレメトリ無効／Webページ取り込み機能は非同梱／Swagger無効／実行時のモデル外部取得なし（オフライン）
- **LANを抜いても・Wi-Fiを切っても通常どおり動く**＝外部依存がないことを顧客自身が確認できる ← 目立たせる
- 推奨: ディスク暗号化（BitLocker等）、Windows/管理者パスワード管理、APIキーの適切な取り扱い

### レイアウト/強調
- 3カラム見出しを「①動作の制約」「②変更しないこと」「③外部に出ない理由」に。
- 左カラム冒頭の**GPU必須**と、右カラムの**「LANを抜いても動く」**を最強調（前者は購入判断の分かれ目、後者は最大の売り）。

## 補足（Codexへの実務メモ）

- 4枚でアイコン言語・配色・フォント・帯の位置を統一し、シリーズとして見えるように。
- 画像内テキストは日本語。英語のコマンド・パス・URL・製品名はそのまま等幅で。
- 生成物は `assets/idea-illustrations/` ではなく、顧客資料として分かる場所（例: `docs/customer-windows/onepagers/`）に保存。ファイル名は次を推奨:
  - `01_install-uninstall`
  - `02_usage-overview`
  - `03-1_usage-upload` / `03-2_usage-backup` / `03-3_usage-manage`（③を3枚にした場合）
  - `04_constraints-notes`
- 迷ったら「非エンジニアの士業が、初見で不安なく理解できるか」を基準に。
