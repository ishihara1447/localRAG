# OTE-RAG トラブルシューティング（Windows 版）

- 対象バージョン: v1.2.0
- 更新日: 2026-07-17

困ったときは、まずこのページの該当する症状を探してください。
コマンドを実行する箇所では、特に断りがない限り
**管理者として実行した PowerShell**（スタートメニューで「PowerShell」を
検索 → 右クリック →「管理者として実行」）を使い、インストール先は
既定の `C:\LocalRAG` として説明します。

## 基本の確認手順（まずこれ）

3 つのサービスが動いているか確認します。

```powershell
Get-Service LocalRAG-*
```

`LocalRAG-Server` / `LocalRAG-Collector` / `LocalRAG-Ollama` の 3 つが
すべて `Running` と表示されれば、システム自体は動いています。
`Stopped` のものがある場合は、次で起動を試してください。

```powershell
cd C:\LocalRAG
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\start.ps1
```

## インストール時のトラブル

> `OTE-RAG-Setup.exe`（ダブルクリック）でインストールした場合、
> インストーラーのログは `C:\ProgramData\OTE-RAG\InstallerLogs`
> に保存されます。エラーで停止したときは、このフォルダの最新の
> ログファイルを提供元にお送りください。以下の各項目のメッセージは、
> ダブルクリック・手動どちらのインストールでも共通です。

### 「不明な発行元」と表示され、Setup.exe が実行できない

現時点の `OTE-RAG-Setup.exe` はコード署名証明書を未取得のため、
Windows SmartScreen が「不明な発行元」と警告する場合があります。
提供元と受け取った 3 ファイル（`OTE-RAG-Setup.exe` /
`OTE-RAG-win64-v1.2.0.zip` / `.zip.sha256`）の組み合わせを IT 担当者が
確認したうえで、「詳細情報」→「実行」を選んでください。

### インストーラが長時間何も表示しない

`[checksum] Verifying package integrity (this can take a while)...` の
表示のあと、**20〜40 分ほど画面に何も出ない時間があります。これは
正常です**（パッケージ全ファイルの破損・改ざん検査を行っています）。
ウィンドウを閉じずにそのままお待ちください。

### 「... failed checksum verification. The package is corrupted.」で停止する

パッケージのコピーや展開の途中でファイルが壊れた・欠けた可能性が
あります。展開済みフォルダを削除し、zip のコピーからやり直して
ください。zip の展開は **エクスプローラーの右クリック →「すべて展開」
または `tar.exe -xf`** で行ってください。PowerShell の
`Expand-Archive` はサポート対象外の展開方法です。再発する場合は
zip ファイル自体の再取得（再コピー）をお試しください。

### 「port XXXX is already in use by process '〇〇'」で停止する（ポート使用中）

OTE-RAG が使う予定のポート（窓口番号）を、別のソフト
（メッセージに表示されたプロセス名）がすでに使っています。

- 表示されたソフトを終了できる場合は、終了してからインストーラを
  再実行してください。
- ポート 3001 の競合で、相手のソフトを止められない場合は、別の
  番号を指定してインストールします。

  ```powershell
  powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\install.ps1 -ServerPort 3002
  ```

  この場合、ブラウザで開く URL も `http://localhost:3002` になります。
- 「port XXXX is held by wslrelay.exe」と表示された場合は、WSL
  （Windows 上の Linux 実行環境）がポートを占有しています。
  管理者 PowerShell で `wsl --shutdown` を実行してから再実行するか、
  IT 担当者にご相談ください。
- ポート 8888 / 11435 は内部処理用で変更できません。占有している
  ソフトを終了してから再実行してください。

### 「nvidia-smi not found」「nvidia-smi failed」で停止する（GPU ドライバ未検出）

NVIDIA GPU のドライバ（GPU を動かすための基本ソフト）が入って
いないか、正しく動作していません。

1. このパソコンに NVIDIA 製 GPU が搭載されているか確認してください
   （搭載されていない場合、本製品の動作要件を満たしません。
   提供元にご相談ください）。
2. 搭載されている場合は、NVIDIA 公式の最新ドライバの導入
   （またはドライバの入れ直し）を IT 担当者に依頼してください。
3. ドライバ導入後にパソコンを再起動し、PowerShell で `nvidia-smi` を
   実行して GPU の情報が表として表示されることを確認してから、
   インストーラを再実行してください。

「WARN: less than 16GB VRAM detected」と表示されて停止した場合は、
GPU のメモリが推奨（16GB）に足りていません。動作保証外のため、
続行するかどうかは提供元にご相談ください。

### 「$InstallRoot\app already exists」「LocalRAG services already exist」で停止する

以前のインストールが残っています。先にアンインストール
（`uninstall.ps1`。既定ではデータは退避・温存されます）を実行して
から、再度インストールしてください。

## 利用中のトラブル

### ブラウザで画面が開けない（接続できない）

1. URL を確認してください。正しくは `http://localhost:3001`
   です（`https://` ではなく **`http://`**。ポート番号を変更して
   いる場合はその番号）。OTE-RAG をインストールした
   **そのパソコンのブラウザ**で開いてください。
2. パソコンを起動した直後の場合は、1〜2 分待ってから再読み込み
   してください（サービスの起動に少し時間がかかります）。
3. それでも開けない場合は「基本の確認手順」でサービスの状態を
   確認し、`Stopped` があれば `start.ps1` で起動してください。
4. 起動してもすぐ `Stopped` に戻る場合は、次項「サービスが起動
   しない」に進んでください。

### サービスが起動しない・すぐ停止する（ログの見方）

動作記録（ログ）が `C:\ProgramData\LocalRAG\logs` に残っています。
エクスプローラーでこのフォルダを開き、**更新日時が新しいファイル**を
メモ帳で開いて確認します。

- `LocalRAG-Server.err.log` / `LocalRAG-Collector.err.log` /
  `LocalRAG-Ollama.err.log` — 各サービスのエラー出力。
  末尾（いちばん下）に直近のエラーが記録されます。
- `LocalRAG-〇〇.out.log` — 通常の動作記録。
- `LocalRAG-〇〇.wrapper.log` — サービスの起動・停止そのものの記録。
  「起動を試みたが失敗した」ときの理由が残ります。

対処の順序:

1. まずパソコンを再起動してみてください（サービスは自動起動します）。
2. 解決しない場合は、上記ログの末尾 20 行ほどを控えて（または
   ファイルごとコピーして）提供元にご連絡ください。

### 文書のアップロードが失敗する・取り込みが終わらない

1. `LocalRAG-Collector` サービス（文書の読み取り担当）が `Running` か
   「基本の確認手順」で確認してください。
2. ファイル形式を確認してください。PDF・Word（.docx）・テキスト
   （.txt）などに対応しています。文字情報を持たないスキャン画像
   だけの PDF は内容を読み取れない場合があります（その場合、
   アップロードできても検索・回答の対象になりません）。
3. 大きなファイルは取り込みに数分かかることがあります。ページ数の
   多い PDF は、少し待ってから画面を再読み込みして状態を確認して
   ください。
4. 特定のファイルだけ失敗する場合は、そのファイルを別名保存
   （PDF の再保存など）してから再試行すると成功することがあります。
5. 解決しない場合は、`C:\ProgramData\LocalRAG\logs` の
   `LocalRAG-Collector.err.log` の末尾を添えて提供元にご連絡ください。

### 回答が遅い・返ってこない

- パソコン起動後の最初の質問は、AI モデルの読み込みのため時間が
  かかることがあります（2 回目以降は改善します）。
- 長い文書や複雑な質問では、検索・再順位付け・回答生成の処理に
  数十秒〜数分かかることがあります。
- 他のソフト（動画編集・ゲーム・別の AI ツールなど）が GPU を
  使っていると大幅に遅くなります。該当するソフトを終了して
  ください。
- 数分待っても何も返らない状態が続く場合は、「基本の確認手順」で
  サービス状態を確認のうえ、パソコンの再起動をお試しください。

### ワークスペースが削除できない

既定で誤削除防止機能が有効なため、プログラム（API）経由の削除は
拒否されます。管理者がブラウザの画面上から削除操作を行って
ください。

## それでも解決しない場合（提供元への連絡）

以下の情報を添えてご連絡ください。原因特定が大幅に早くなります。

1. 症状と、発生した日時・直前に行った操作
2. サービスの状態（下記の出力をコピー）

   ```powershell
   Get-Service LocalRAG-*
   ```

3. ログフォルダ `C:\ProgramData\LocalRAG\logs` のうち、更新日時が
   新しいファイル（特に `*.err.log` と `*.wrapper.log`）
4. バージョン情報: インストール先（`C:\LocalRAG`）にある
   `versions.lock` ファイル

**注意: ログやバックアップを社外へ送る前に、所内の情報管理ルールを
確認してください。** ログには文書のファイル名などが含まれる場合が
あります。
