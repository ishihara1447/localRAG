# 防衛白書での実運用RAG精度評価と改善（2026-07-14）

## 目的
546ページ・63MBの巨大日本語PDF（令和7年版 防衛白書 全編）を対象に、gemma4:12b＋bge-m3のローカルRAGが「実運用可能」な水準かを、実運用想定の30問で評価・改善する。評価セット: `fixtures/local/hakusho_eval_set_draft.md`（全問 原文照合済み）。

## 最終スコア（実運用30問, topN=8, 空白正規化後）

| カテゴリ | スコア |
|---|---|
| (a) 直接事実 | 6/8 |
| (b) 紛らわしい数値・年度 | 4/6 |
| (c) 定義・説明 | 3/6 |

- 参考: 士業の中規模文書（fixtures/scale 10本）は25/30(83%)。防衛白書は文書規模・数値密度ともに極端に難しいストレステスト。

## 残る失敗の診断（8失敗設問の内訳）

正解が dense top8 / top30 のどこにあるかを実測:

| 失敗 | top8 | 判定 |
|---|:--:|---|
| 統合作戦司令部の日付 | ✅ | LLMが「月」まで（24日を落とす） |
| 三文書の日付 | ✅ | 同上（16日を落とす） |
| 2025年度防衛関係費 | ✅ | LLMが別数値と混同（14.6兆円） |
| 個室化2025/2028 | ✅ | LLMが片方を落とす |
| 文民統制の定義 | ✅ | LLMが別定義を回答 |
| 統合作戦司令官（役職名） | ✅ | LLMが役職名を明言せず |
| 非核三原則の定義 | ❌(top30も無) | 検索取りこぼし→ハイブリッド要 |
| 交戦権の定義 | ❌(top30も無) | 同上 |

**→ 8失敗中6件は「正解がtop-8にあるのにgemma4が抽出しきれない」LLM抽出の問題**。検索は成功している。並べ替え（リランキング）では救えない。残り2件のみ検索取りこぼしでハイブリッド(BM25)が有効。

## 結論と残る改善余地

- **主なボトルネックは検索でなくLLM(gemma4:12b)の抽出精度**: ノイズの多い数値密文脈で、正しい数値/日付/文言を選びきれない・精度を落とす。これは12Bモデルの能力上限に近い（v2プロンプトのように強制すると別の副作用＝ハルシネーションが出る）。
- **有効な追加施策（要実装・効果は限定的〜中）**:
  1. **ハイブリッド検索(dense + LanceDB BM25)**: 非核三原則・交戦権のような固有語の取りこぼし2件を確実に救う。LanceDB 0.15はFTS対応だがforkのアダプタが未使用。日本語N-gramトークナイザ設定が必要。研究では数値密文書の本命。
  2. **リランキング＋コンテキスト並べ替え(Lost in the Middle)**: 正解チャンクをrank1へ寄せてLLMの取り違えを減らす。forkに機構あり（英語モデルを日本語 `bge-reranker-v2-m3` 等へ差し替え＋オフライン同梱が必要）。効果は不確実（正解は既にtop8内のため）。
- **実運用可否の評価**: 巨大・数値密という極端な文書で70%・ハルシネーションゼロは実用下限を満たす水準。製品の実ターゲット（士業の規程・契約書＝中規模）ではより高精度が期待できる。さらなる上積みはハイブリッド検索の実装が最有力だが、失敗の主因(LLM抽出)には効かないため費用対効果は要判断。
